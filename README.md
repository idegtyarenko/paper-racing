# Paper Racing 🏁

**Гонки по клеточкам** — та самая игра из школьной тетради, где болиды ездят по клеточкам.

**[Играть!](https://idegtyarenko.github.io/paper-racing/)**

[![Ask DeepWiki](https://deepwiki.com/badge.svg)](https://deepwiki.com/idegtyarenko/paper-racing)

> 🧑‍🍼 Это вайбкод-проект, который мы делаем с младшим сыном: придумываем правила, рисуем трассы, играем и тут же чиним то, что не понравилось.

## Фишки

- ✏️ **Свои трассы** — рисуешь любую конфигурацию, а не выбираешь из готовых.
- 🧠 **Две модели физики** — «простая» (классические клеточные правила инерции ±1) и «реалистичная» («круг сцепления»: нельзя одновременно резко тормозить и резко поворачивать — появляется настоящая гоночная траектория с апексом).
- 🌐 **Онлайн-мультиплеер** — совместная игра через бэкенд на [Supabase](https://supabase.com).
- 🤖 **Синглплеер с ботами**.
- 📱 **PWA** — ставится на домашний экран и работает как приложение, в том числе офлайн (локальная игра).
- ⚙️ **Настройки заезда** — штраф за аварию (статический или «по скорости»), модель физики, очередность ходов.

## Технологии

Чистый **TypeScript** без фреймворка, рендер на `<canvas>`, сборка на **Vite**, офлайн — через `vite-plugin-pwa`. Онлайн — **Supabase** (realtime + REST). Тесты — **Vitest**.

## Запуск

```bash
npm install
npm run dev        # локальный dev-сервер
```

Для онлайн-режима скопируй `.env.example` → `.env` и подставь ключи своего проекта Supabase. Без них сборка проходит и локальная игра работает — просто скрыты онлайн-входы.

```bash
npm run build       # typecheck + тесты + прод-сборка
npm run preview     # предпросмотр прод-сборки

npm run test        # тесты (vitest, разовый прогон)
npm run test:watch  # тесты в watch-режиме
npm run typecheck   # только проверка типов (tsc --noEmit)

npm run format      # отформатировать всё (prettier)
npm run format:check # проверить форматирование без правок
npm run typo        # расставить неразрывные пробелы в строках UI (src/strings.ts)
npm run typo:check  # проверить неразрывные пробелы без правок
```
